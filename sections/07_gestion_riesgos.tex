\section{Gestión de Riesgos}

\subsection{Metodología de Evaluación de Riesgos}
La gestión de riesgos sigue un enfoque cuantitativo basado en dos parámetros (Escala 1-5):
\begin{itemize}
    \item \textbf{Probabilidad (P):} Frecuencia estimada de ocurrencia (1: Muy rara, 5: Frecuente).
    \item \textbf{Impacto (I):} Severidad de la consecuencia para la misión (1: Insignificante, 5: Catastrófica).
    \item \textbf{Criticidad (C):} Producto de $P \times I$. (1-8: Bajo, 9-15: Medio, 16-25: Alto).
\end{itemize}

\subsection{Monitoreo y Control de Riesgos}
El sistema implementa un sistema de salud (\textit{Health Monitor}) en ROS 2 que evalúa continuamente estas métricas y dispara las estrategias de contingencia de forma autónoma sin intervención humana directa, minimizando el tiempo de respuesta ante fallos críticos.

\section{Apéndices}

\subsection{Glosario de Términos}
\begin{itemize}
    \item \textbf{CSP:} CubeSat Space Protocol (Transporte de red ligero para sistemas embebidos).
    \item \textbf{EKF:} Extended Kalman Filter (Algoritmo de estimación de estado).
    \item \textbf{SBC:} Single Board Computer (Computadora de placa única, ej. Raspberry Pi).
    \item \textbf{FDIR:} Fault Detection, Isolation, and Recovery.
\end{itemize}

\subsection{Trazabilidad de Requerimientos (Resumen)}
Todos los riesgos técnicos (RT) están mapeados a requerimientos de seguridad y FDIR definidos en la Sección 3.2, asegurando que cada amenaza identificada posea una contraparte de diseño técnico.
