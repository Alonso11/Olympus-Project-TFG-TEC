\section{Validación y Verificación}

\subsection{Estrategia de Verificación y Validación (V\&V)}
La verificación asegura que el sistema se construyó correctamente (cumple el SRS), mientras que la validación asegura que se construyó el sistema correcto (cumple los objetivos de la misión). Se emplean cuatro métodos estándar:

\begin{itemize}
    \item \textbf{Análisis (A):} Uso de modelos, simulaciones o cálculos para verificar el cumplimiento.
    \item \textbf{Inspección (I):} Verificación visual de componentes, planos o código fuente.
    \item \textbf{Demostración (D):} Operación funcional sin necesidad de instrumentación precisa para validar un comportamiento.
    \item \textbf{Prueba (T):} Medición cuantitativa bajo condiciones controladas con instrumentación.
\end{itemize}

\subsection{Procedimientos de Prueba (Test Procedures)}

\subsubsection{TP-01: Prueba de Movilidad y Pendiente (STR/NAV)}
\begin{itemize}
    \item \textbf{Preparación:} Rover en rampa de 15$^\circ$ con regolito simulado. Batería > 50\%.
    \item \textbf{Ejecución:} Comando de avance lineal de 2 metros.
    \textit{Resultado Esperado:} El rover alcanza la cima sin intervención y sin sobrecalentamiento de motores.
\end{itemize}

\subsubsection{TP-02: Evasión Dinámica de Obstáculos (NAV/CDH)}
\begin{itemize}
    \item \textbf{Preparación:} Entorno con obstáculos de 12 cm de altura no mapeados.
    \item \textbf{Ejecución:} Envío de un \textit{goal} de navegación a 5 metros de distancia.
    \textit{Resultado Esperado:} El sistema genera una ruta local que rodea el obstáculo manteniendo una distancia de seguridad.
\end{itemize}

\subsubsection{TP-03: Validación de Modos de Fallo y FDIR (CDH/PWR)}
\begin{itemize}
    \item \textbf{Preparación:} Rover en modo nominal. Simulación de desconexión de antena o baja batería.
    \item \textbf{Ejecución:} Interrupción del latido de la GCS o forzado de lectura de voltaje bajo.
    \textit{Resultado Esperado:} El rover detiene motores y publica estado ``SAFE\_MODE'' en < 10 s.
\end{itemize}

\subsubsection{TP-04: Caracterización del Enlace CSP/RF (COM/CDH)}
\begin{itemize}
    \item \textbf{Preparación:} GCS a 30 metros del rover. Monitor de red activo.
    \item \textbf{Ejecución:} Envío de ráfaga de 100 paquetes de telemetría.
    \textit{Resultado Esperado:} Latencia media $\le$ 500 ms y tasa de pérdida de paquetes < 1\%.
\end{itemize}

\subsection{Criterios de Aceptación Final}
El sistema Olympus se considerará apto para la misión si:
\begin{enumerate}
    \item Se han verificado satisfactoriamente todos los requerimientos marcados como \textbf{Críticos}.
    \item No existen fallos de segmentación o reinicios no controlados en ROS 2 durante 1 hora de operación.
    \item La integridad física de la estructura se mantiene tras completar el ciclo de TPs.
\end{enumerate}
