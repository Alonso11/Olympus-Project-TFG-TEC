\section{Especificación de Requerimientos del Sistema}

Este capítulo detalla los requerimientos formales del sistema Olympus. Siguiendo los estándares de ingeniería de sistemas, cada requerimiento utiliza el verbo auxiliar \textbf{``deberá''} para denotar obligatoriedad.

\subsection{Requerimientos de Capacidad (Funcionales)}

\subsubsection{Navegación y Control (GNC)}
\begin{itemize}[label=---]
    \item \textbf{RF-GNC-01:} El sistema \textbf{deberá} calcular rutas de navegación autónoma evitando obstáculos mayores a 10 cm de altura.
    \item \textbf{RF-GNC-02:} El rover \textbf{deberá} ser capaz de realizar una parada de emergencia de forma autónoma ante la pérdida de datos del sensor LiDAR.
    \item \textbf{RF-GNC-03:} El sistema \textbf{deberá} mantener una precisión de localización con un error máximo de $\pm$ 50 cm en un área de 10x10 metros.
\end{itemize}

\subsubsection{Movilidad}
\begin{itemize}[label=---]
    \item \textbf{RF-MOV-01:} El tren de rodaje \textbf{deberá} permitir el avance en pendientes de hasta 15$^\circ$ sobre regolito simulado.
    \item \textbf{RF-MOV-02:} El rover \textbf{deberá} alcanzar una velocidad máxima nominal de 10 cm/s en terreno llano.
\end{itemize}

\subsubsection{Gestión de Energía}
\begin{itemize}[label=---]
    \item \textbf{RF-ENE-01:} El subsistema de potencia \textbf{deberá} monitorizar el voltaje y la corriente de las baterías con una frecuencia mínima de 1 Hz.
    \item \textbf{RF-ENE-02:} El sistema \textbf{deberá} iniciar un protocolo de ``Modo Seguro'' cuando el estado de carga (SoC) sea inferior al 15\%.
\end{itemize}

\subsection{Requerimientos Físicos y Ambientales}
\begin{itemize}[label=---]
    \item \textbf{RE-ENV-01:} El sistema \textbf{deberá} operar nominalmente en un rango de temperatura de -20$^\circ$C a +50$^\circ$C (condiciones análogas).
    \item \textbf{RE-PHY-01:} La masa total del rover no \textbf{deberá} exceder los 15 kg, incluyendo la carga útil científica.
\end{itemize}

\subsection{Atributos de Calidad (No Funcionales)}

\subsubsection{Confiabilidad y Seguridad (RAMS)}
\begin{itemize}[label=---]
    \item \textbf{RNF-REL-01:} El sistema \textbf{deberá} tener un tiempo medio entre fallos críticos (MTBCF) de al menos 100 horas de operación continua.
    \item \textbf{RNF-SAF-01:} El rover \textbf{deberá} entrar en estado de reposo (Safe Mode) si se pierde el enlace de mando por más de 10 segundos.
\end{itemize}

\subsubsection{Mantenibilidad}
\begin{itemize}[label=---]
    \item \textbf{RNF-MTN-01:} El software del rover \textbf{deberá} permitir actualizaciones remotas (OTA) a través de la interfaz de comunicaciones.
\end{itemize}

\subsection{Interfaces Externas}

\subsubsection{Interfaces de Usuario (HMI)}
\begin{itemize}[label=---]
    \item \textbf{RI-USR-01:} La estación de control \textbf{deberá} mostrar una telemetría visual en tiempo real que incluya posición, velocidad y estado de batería.
\end{itemize}

\subsubsection{Interfaces de Software}
\begin{itemize}[label=---]
    \item \textbf{RI-SW-01:} El sistema \textbf{deberá} utilizar el middleware ROS 2 (Humble o superior) para la comunicación entre nodos internos.
\end{itemize}
